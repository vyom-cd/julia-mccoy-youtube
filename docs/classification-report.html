Root Causes

Why 564 comments changed classification

  • good_point is a black hole — any comment >30 chars that doesn't match another pattern falls into good_point. The old classifier (classify_all.py) had the same threshold at 30 chars but had more aggressive pattern matching for complaints, spam, and testimonials that caught comments before they fell through.
  • question is too aggressive — any comment containing "?" with >15 chars becomes a question. Many rhetorical questions in complaints and opinions get reclassified. Old classifier had no secondary "?" catch.
  • Complaint patterns too narrow — the old classify_all.py had 20+ complaint terms that weren't merged (e.g., "this is crap", "sound funny", "ain't it", "slop", "delusion"). Some were merged but several edge cases are missing.
  • Spam regex misses — the numerology spam pattern own:\d+.*one:\d+.*from:\d+ required 3 parts in the old classifier but only 2 in the new one. Some numerology spam still falls through to good_point because the patterns aren't specific enough.
  • Testimonial length threshold changed — old: 150 chars. New: 150 chars for praise, but 50 chars for thanks. "Very interesting." (17 chars, no praise keyword match) falls to good_point instead of testimonial.
Recommended Fixes

3 changes to get agreement above 85%

  • 1. Raise the good_point floor to 80+ chars — comments under 80 chars that don't match any pattern should be "other", not "good_point". This alone fixes ~200 misclassifications.
  • 2. Add missing complaint patterns — merge the remaining terms from classify_all.py that were dropped: "sound funny", "is crap", "slop", "ai clone", "nonsense", "bs", "ai script", explicit profanity patterns.
  • 3. Widen testimonial catch — "Very interesting" and similar short positive reactions (under 30 chars) should be testimonial. Add "interesting", "nice", "cool" as short-form testimonial triggers.
